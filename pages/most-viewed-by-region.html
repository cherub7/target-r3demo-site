What’s most viewed in your region

Using this page — Changing region reloads the most-viewed list below. Click a title to generate a view event for the selected region. The list shows popular titles for that region (aggregated across visitors)—not personalized to you.

How this demo works — Pick a region, simulate catalog views, then load a most viewed list for that region (based on aggregated views, not a personal “for you” feed).

View events and the recommendation request both send the mbox parameter recsRegion.

Step 1 — Catalog — Each title click generates a view event on item-view-mbox with entity.* and the current recsRegion.

Step 2 — Recommendations — The recs panel calls Target with mbox to surface what’s most viewed for recsRegion. The region from the sticky bar is passed as recsRegion.

Target setup — Recommendations are grouped by region. In Target, profile script recsAttributeRegion can be return mbox.param("recsRegion"); (align with your Recs profile attribute key). Add ?recsDebug=1 to the URL to log the raw offer in the console.
