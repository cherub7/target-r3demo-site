How this demo works

Criteria type — "Top Sold by Item Attribute". utu groups purchase events by entity.custom2 (region) and computes a separate top-sellers list per value: India, US, UK.

Delivery — Each strip calls mbox r3demo-top-sold-by-item-attribute with recsRegion=<value>. A profile script maps that param to user.recsAttributeRegion, which the criteria uses as the ER lookup key at r3edge.

Catalog Buy button — Sends a purchase event on item-purchase-mbox with the book's own region, so it contributes to the correct regional top-sellers list.

Debug — Add ?recsDebug=1 to log raw offers to the console.
